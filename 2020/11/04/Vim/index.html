Vim

Operator + Motion = Action

  • 一次**操作（Action）由一个操作符（operator）后接一个动作命令（motion）**组成。

  • 当一个操作符命令被来连续调用两次时，它会作用于当前行。例如：

    • dd：删除当前行
    • >>：缩进当前行
    • gUgU, gUU : 当前行大写

  • 自定义操作符（operator）与动作（motion）命令：一些插件

    • commentary.vim : gc{motion}，其中gcc注释当前行。
    • textobj-entire : 定义了新的motion，针对整个文件
    • 自己编写自定义操作符或动作：
      • :h :map-operator
      • :h omap-info

text-objects

Text objects are two characters used after an operator (‘d’, ‘y’, ‘c’ etc.) to select a range to operate on. The first character is either ‘a’ (for “an object”, including white space) or ‘i’ (for “inner” object, without surrounding white space, or only the white space). The second character denotes the object type and is one of the following:

word, WORD, sentence, paragraph,**
" or ’ or ` - a ", ', or ` quoted string
{ , }, or B - A { } block
(, ), or b - A ( ) block
[ or ]
< or > - A < > block
t - A HTML or XML tag block

word w

A word consists of a sequence of letters, digits and underscores, or a sequence of other non-blank characters, separated with white space (spaces, tabs, ). An empty line is also considered to be a word.

Use ‘w’, ‘e’, and ‘b’ to navigate words.

Special case: “cw” is treated like “ce” if the cursor is on a non-blank. This is because “cw” is interpreted as change-word, and a word does not include the following white space.

WORD W

A WORD consists of a sequence of non-blank characters, separated with white space. An empty line is also considered to be a WORD.

Use ‘W’, ‘E’, and ‘B’ to navigate WORDs.

Special case: “cW” is treated like “cE” if the cursor is on a non-blank. This is because “cW” is interpreted as change-WORD, and a WORD does not include the following white space.

paragraph p

A paragraph begins after each empty line. A blank line (only containing white space) is NOT a paragraph boundary.

Use ‘{’ and ‘}’ to navigate paragraphs. Paragraph text objects are denoted with ‘p’.

sentence s

A sentence is defined as ending at a ‘.’, ‘!’ or ‘?’ followed by either the end of a line, or by a space or tab. Any number of closing ‘)’, ‘]’, ‘"’ and ‘’’ characters may appear after the ‘.’, ‘!’ or ‘?’ before the spaces, tabs or end of line. A paragraph boundary is also a sentence boundary.

Use ‘(’ and ‘)’ to navigate sentences. Sentence text object is denoted by ‘s’.

替换

语法: :[addr]s/源字符串/目的字符串/[option]

  • [addr] 表示检索范围,如:

    • 1,n 表示从第1行到n行
    • % 表示整个文件,同1,$
    • .,$ 表示从当前行到文件尾
    • 省略时表示当前行

  • s 表示替换操作,其为substitute的缩写

  • [option] : 表示操作类型,如:

    • g globe,表示全局替换
    • c confirm,表示进行确认
    • p 表示替代结果逐行显示(Ctrl + L恢复屏幕)
    • i ignore,不区分大小写
    • 省略时仅对每行第一个匹配串进行替换

  • 如果在源字符串和目的字符串中出现特殊字符,如’/‘,’<‘,’>‘,’,'等需要前面加反斜杠\进行转义

  • & to repeat, u to reverse

vim多行缩进或缩出方法

Visual mode

按v进入visual状态，选定多行，用>缩进，用<缩出

缩进

单行缩进

在要缩进的那一行按下>>, 即连按两下>。

多行缩进

在要缩进的起始行按下n>>，n是要缩进的行数。

多行缩出

单行缩出

在要缩出的那一行按下<<, 即连按两下<。

多行缩出

在要缩出的起始行按下n<<，n是要缩出的行数。

撤销与重做

u

Ctrl-r

U

对于撤销/重做 (undo/redo) 来说，“U” 命令和其他命令是同等对待的。所以 “u” 可以

撤销一个 “U” 命令、‘CTRL-R’ 也可以重做对 “U” 命令的撤销。当 “U”、“u” 和

‘CTRL-R’ 三个命令混合使用时，你将注意到 “U” 命令决意将被该行还原到前一个 “U”

命令之前的状态。这或许会令你困惑。多试试，你会习惯的。

“U” 命令总是把缓冲区标记为已改变。即使它把修改过的缓冲区重做到未改变的状态也是

如此。为此，只有使用 “u” 命令，才能令缓冲区正确地回复到未改变的标记。

控制撤销操作的粒度

control the granularity of the undo command

  • 按照喜好，每次思考后停顿（可能是一句、一个词、一段等）的过程都退出insert mode，使其成为一个撤销块。这样更好控制撤销的粒度。

  • 另起一行时也可以用<Esc>o来代替保持insert mode下<CR>。原因与上同理。

  • Moving Around in Insert Mode Resets the Change 每次在插入模式中使用上下左右光标移动时，都会产生一个新的撤销块

    When I said that the undo command would revert all characters entered (or deleted) during a trip into Insert mode and back, I was glossing over a small detail. If we use the , , , or cursor keys while in Insert mode, a new undo chunk is created. It’s just as though we had switched back to Normal mode to move around with the h , j , k , or l commands, except that we don’t have to leave Insert mode. This also has implications on the operation of the dot command.

Insert Mode

核心是在插入模式如何使用一些在普通模式能使用的操作，从而简化插入过程。

  • 从insert mode返回normal mode:

    • Esc: 已经和Caps Lock进行了交换，要逐步习惯这种布局
    • <Ctrl-[>: 比传统Esc距离更近，但替换后意义不大

  • 在插入模式中可以进行的操作

    • 删除操作：

      • <Ctrl-h> : 删除前一个字符

      • <Ctrl-w> : 删除前一个单词

      • <Ctrl-u> : 删至行首

      这些在bash shell中也是可以通用的。

    • 执行一条普通模式下的指令：<Ctrl-o>

      Insert-Normal Mode

      非常重要，在插入模式中进入插入-正常模式，允许你在不离开插入模式的情况下执行一条普通模式下的指令。

    • 粘贴操作：<Ctrl-r>{register}

      将{register}指定的内容粘贴。

Register

以 " 为开头的名字代表寄存器。

  • 寄存器的分类：

    • the unnamed register: ""

      默认的寄存器

    • the named registers: "a - "z

      "A - "Z当输入大写字母时，会 append 而非 replace

    • the numbered registers: "1 - "9

      删除的内容会依次存在编号的寄存器中，超过9个就丢弃最老的；

    • the Black Hole register: "_

      相当于一个garbage dump

    • "- 等其他见 Practical Vim

  • :reg : 命令行输入 reg 以查询当前所有寄存器的名字及其中的内容；

  • 以寄存器名为开头来指定命令的结果存储到哪，如："2p, "adw

level 12

text-objects见下

  • { } [count] paragraphs forward/backward

    Paragraphs are seperated by empty lines.

    { : Move the cursor to the empty line before the current paragraph, or before the previous paragraph if the cursor isn’t in a paragraph.

    } : Move the cursor to the empty line after the current paragraph, or after the next paragraph if the cursor isn’t in a paragraph.

  • ( ) [count] sentence forward/backward

  • [ ] go to [count] previous (or next) unmatched ‘{’ (or ‘)’) starting at, but not including, cursor position.

  • ia

  • . : Repeat the last simple change. Without a count, the count of the last change is used. If you enter a count, it will replace the last one.

    If the last change included a specification of a numbered register, the register number will be incremented.

    Does not repeat a command-line command.

level 14

  • H : To line [count] from first line on the window (High) on the first non-blank character (linewise), without scrolling the screen.

  • L

  • M

  • :<count> : 可以同样起到跳转至某行的作用

  • nu, nonu

    Show the line number in front of each line in the text. Off by default.

    Makes it really easy to jump to a specific line using ‘:’ and the line number, or the line number followed by G or gg.

    :set nu - shows line numbers.
    :set nonu - hides line numbers.
    :set nu! - toggle line numbers.
    :set nonu!- toggle line numbers.

    如果想要默认进入Vim即显示行号，则需要修改~/.vimrc配置文件，在配置文件中写入set nu即可。

  • ：| : To column [count] in the current line.

  • <Ctrl-a>, <Ctrl-x> 把当前光标之上或之后的数值加上/减去[count]，不带次数默认为1

  • vim中把以0开头的数字解释为八进制而非十进制

    • 例如 007 + 001 = 010
    • vimrc中加入set nrformats= 会让vim把所有数字都当成十进制，无论是否0开头。
