15-213: Shell Lab

Introduction

A Tiny Unix Shell with job control.

Goals

已有的轮子：

剩余需要实现的：

  • eval: Main routine that parses and interprets the command line. [70 lines]

    Shell的入口，命令行的读入与解析。

  • builtin_cmd: Recognizes and interprets the built-in commands: quit, fg, bg, and jobs. [25 lines]

    识别并执行Shell的几个内置命令。

  • do_bgfg: Implements the bg and fg built-in commands. [50 lines]

    实现bg与fg，即命令在shell中执行的前台后台切换。

  • waitfg: Waits for a foreground job to complete. [20 lines]

    shell等待一个前台工作完成。

三个 Signal Handler:

  • sigchld_handler: Catches SIGCHILD signals. [80 lines]

  • sigint_handler: Catches SIGINT (ctrl-c) signals. [15 lines]

  • sigtstp_handler: Catches SIGTSTP (ctrl-z) signals. [15 lines]

其他要求：

  • Shell 的提示标记符为 "tsh> "

  • 用户键入的命令行应包括命令名称和 0 个或多个参数，所有参数以一个或多个空格分隔。如果 name 是一个内置命令，则 tsh 应该立即处理它并等待下一个命令行。否则，tsh 假定该名称是一个可执行程序的路径，并在在一个初始子进程的上下文中加载并运行

  • tsh 不需要支持管道（|）或 I/O 重定向（< 和 >）

  • 键入 ctrl-c（ctrl-z）应该会发送一个 SIGINT（SIGTSTP）信号给当前的前台作业以及它的子进程。如果没有前台作业，则该信号应该没有任何作用

  • 如果命令行以 ＆ 结束，则 tsh 应该在后台运行该作业。否则，它应该在前台运行

  • 每个作业都可以由一个进程 ID（PID）或一个作业 ID（JID）标识，该 ID 是一个由 tsh 分配的正整数。 JID应该在命令行上以前缀 “％” 表示。例如，“％5” 表示 JID 5，“ 5” 表示 PID 5

  • tsh 应该支持以下内置命令：

    • quit：终止 tsh 程序。
    • jobs：列出所有后台作业
    • bg ：向作业发送 SIGCONT 信号来重新启动 ，然后在后台运行
    • fg ：向作业发送 SIGCONT 信号来重新启动 ，然后在前台运行

  • tsh 必须回收所有的僵死进程

wait_fg

do_bgfg中的一部分，等待前台进程执行完毕。实现方式：挂起/阻塞父进程（即Shell），直至前台进程执行完毕发来信号SIGCHLD。

do_bgfg

内置命令的实现。功能是发送SIGCONT信号使其重启。

buildtin_cmd

eval的实现中需要判断内置指令。argv[0]中默认存放程序名，对其检查并判断是哪个内置指令即可，无复杂逻辑。

eval

After the fork, but before the execve, the child process should call setpgid(0, 0), which puts the child in a new process group whose group ID is identical to the child’s PID. This ensures that there will be only one process, your shell, in the foreground process group. 从而防止子进程不会跟Shell同一组
